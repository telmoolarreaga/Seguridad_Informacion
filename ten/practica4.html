Práctica 04

Romper el HASH de la clase - Cracker a Fuerza Bruta

Criptografía Python SHA-256 Brute Force
01

Ejecución y Resultados

Reto A: 4 caracteres
> Target: 3cdea1b06e8f4c08213ca4cbd702a27871914c971ee226cf9debf88b8435c375
Contraseña encontrada: vini
Intentos: 374851
Tiempo: 0.48151254653930664 segundos

Sabíamos que eran 4 caracteres minúsculas sin números ni caracteres especiales.

import hashlib
import itertools
import string
import time

# Leer el hash del archivo (solo lectura, no se sobrescribe)
archivo = open("hash_guardado.txt", "r")
hash_objetivo = archivo.read().strip()
archivo.close()

caracteres = string.ascii_lowercase  # letras a-z
intentos = 0

inicio = time.time()

# Generar todas las combinaciones de 4 letras
for combinacion in itertools.product(caracteres, repeat=4):
    password = "".join(combinacion)

    # Calcular hash SHA-256
    hash_generado = hashlib.sha256(password.encode()).hexdigest()

    intentos += 1

    if hash_generado == hash_objetivo:
        fin = time.time()
        print("Contraseña encontrada:", password)
        print("Intentos:", intentos)
        print("Tiempo:", fin - inicio, "segundos")
        break
Reto B: 7 caracteres con patrón
> Target: 4a00382efb5acb17c52a699364e65fc0189a4235e4fb467eb085cfa565cfeaf4
Contraseña encontrada: cr7goat
Intentos: 114940
Tiempo: 0.15668964385986328 segundos

Sabíamos que eran 7 caracteres, en minúsculas un solo número y que contenía "cr7".

import hashlib
import itertools
import string
import time

# Leer hash
with open("hash_nuevo.txt", "r") as f:
    hash_objetivo = f.read().strip()

letras = string.ascii_lowercase
inicio = time.time()
intentos = 0

# Como CR7 ya está en orden, necesitamos completar los 7 caracteres
# El patrón: 'c', 'r', '7' + 4 letras
# Vamos a colocar las 4 letras en las posiciones restantes (total 7)

# Posiciones posibles para CR7
# c en pos i, r en pos j>i, 7 en pos k>j
# luego las 4 letras en las otras posiciones

# Para simplificar, vamos a asumir CR7 al principio y generar todas combinaciones de 4 letras después
for letras_restantes in itertools.product(letras, repeat=4):
    password = 'cr7' + ''.join(letras_restantes)
    hash_generado = hashlib.sha256(password.encode()).hexdigest()
    intentos += 1

    if hash_generado == hash_objetivo:
        print("Contraseña encontrada:", password)
        print("Intentos:", intentos)
        print("Tiempo:", time.time() - inicio, "segundos")
        exit()

Lógica del Algoritmo

a. El proceso de Brute Force

El script utiliza la librería itertools.product para generar el espacio de búsqueda. Los pasos clave son:

  • Lectura del hash objetivo desde el archivo.
  • Generación de combinaciones de string.ascii_lowercase
  • Codificación en UTF-8 y cálculo del hashlib.sha256()
  • Comparación hexadecimal para detener el bucle al hallar el match.
b. Complejidad Computacional

Para una clave de 4 letras minúsculas, existen 26^4 combinaciones posibles. Aunque parece mucho, una computadora puede procesarlo en menos de un segundo.

Si añadiéramos números, mayúsculas y caracteres especiales, el tiempo crecería exponencialmente, demostrando la importancia de la entropía en las contraseñas.

c. Fuerza Bruta vs. Información

    En el primer reto, el ataque fue puramente secuencial sobre 4 posiciones. En el segundo, a pesar de tener 7 caracteres, el conocimiento previo (el patrón "cr7") permitió optimizar el script para probar muchas menos combinaciones (114.940 frente a 94.931.877.133 posibles sin patrón).

VA

Valoración

Utilidad

El uso de patrones conocidos reduce el tiempo de búsqueda drásticamente, esto es algo que desconociamos. Para el reto final de clase esta practica puede ser una de las más importantes.

Dificultad

Dificultad baja-media, centrada en la optimización de bucles e iteraciones en Python.
